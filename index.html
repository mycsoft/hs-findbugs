使用说明

红松代码检查器-Findbugs,是专用于红松平台开发的代码检查工具.本工具基于Findbugs 3.0.1进行开发.

Error Code

  • HS_SERVICE_SQL: 在非Dao中发现SQL语句.

    在Action,Form,Service等非Dao代码中,直接调用SQL语句,会引起扩展性问题.

  • HS_SYSTEM_OUT_PRINT: 在代码中使用了System.out.println.

    在代码中使用了System.out.println.应当使用日志方式.如log.info("......")

  • HS_EXCEPTION_PRINT: 在代码中将异常直接输出.

    代码中将异常直接输出.如ex.printStackTrace(). 应当使用日志方式.如log.warning("......",ex).

  • HS_PRJACC_UNSUMMED:使用项目台帐时,未正确触发汇总.

    代码中如调用到创建项目台帐的方法.应当在其后调用相应的汇总方法.PrjAccService.sumPrjAcc();

  • HS_MORE_RETURN: 方法中使用了了过多的return.

    一个方法中尽量只使用一个返回语句.

  • HS_ERROR_EXCEPTION: 异常使用错误.

    方法中只能在Service与Action中抛出异常.且只能抛出指定的系统内的异常. 如: `throw new ServiceException(GlobalExceptions.NAME_EXIST,new Object[]{"基础数据分类"})';

  • HS_MORE_LOOP: 代码中循环嵌套不可以超过3层.

    代码中循环嵌套不可以超过3层.这对代码的可读性,与程序的性能都有很大的影响.

  • HS_LOOP_SQL: 在循环中调用数据库连接,严重影响性能.

  • HS_ILLEGAL_CALL_DAO: 非法的Dao调用.

    非法调用了Dao方法,破坏了代码结构.应该通过对应Service方法进行调用.

  • HS_UNUSED_METHOD: 未使用到的方法.建议删除此方法.
